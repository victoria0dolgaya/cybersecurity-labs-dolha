Лабораторна робота №4
Навчальна модель цифрового підпису

Дисципліна: Безпека програм та даних

📌 Опис
Дана програма реалізує спрощену навчальну модель цифрового підпису файлів.

Вона демонструє основні принципи:
формування приватного та публічного ключів;
створення цифрового підпису на основі хешу файлу;
перевірку дійсності підпису;
виявлення підробки при зміні вмісту файлу.

Програма має демонстраційний характер і призначена для навчальних цілей.

🎯 Мета роботи
ознайомитися з принципами цифрового підпису;
реалізувати залежність підпису від вмісту файлу та ключів;
дослідити механізм перевірки цілісності даних;
показати неможливість підробки підпису без зміни файлу або ключа.

⚙️ Функціональні можливості
Програма дозволяє:
формувати приватний ключ на основі персональних даних користувача;
обчислювати публічний ключ, пов’язаний із приватним;
створювати цифровий підпис для файлу;
перевіряти коректність підпису;
імітувати підробку шляхом зміни даних файлу;
працювати в інтерактивному консольному режимі.

▶️ Як запустити програму
🔹 Вимоги
Python 3.x
🔹 Запуск
Збережіть код у файл, наприклад digital_signature_demo.py
Запустіть програму:
python digital_signature_demo.py

Дотримуйтесь інструкцій у консолі.

🧠 Логіка роботи програми
sha256() — обчислює SHA-256 хеш для заданих даних.
build_private_key() — формує приватний ключ з персональних даних користувача.
build_public_key() — створює публічний ключ як похідне від приватного.
create_signature() — створює цифровий підпис файлу на основі його хешу та приватного ключа.
check_signature() — перевіряє дійсність підпису шляхом відновлення хешу та порівняння з поточним.
main() — реалізує меню та взаємодію з користувачем.

Приватний ключ (hex): a1b2c3...

У разі зміни файлу:

⚠️ Примітка
Даний алгоритм не є повноцінною реалізацією цифрового підпису і не повинен використовуватися у реальних системах безпеки. Програма призначена виключно для демонстрації базових криптографічних принципів у межах навчального курсу.
