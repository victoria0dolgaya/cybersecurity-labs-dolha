Навчальна модель цифрового підпису

Дисципліна: Безпека програм та даних

📌 Опис
Дана програма реалізує спрощену навчальну модель цифрового підпису текстових повідомлень з використанням алгоритму RSA та хеш-функції SHA-256.

Програма демонструє основні принципи цифрового підпису:
генерацію приватного та публічного ключів;
створення цифрового підпису на основі хешу повідомлення;
перевірку дійсності підпису;
виявлення зміни даних після підписання.

Реалізація має навчальний характер та використовується для демонстрації базових криптографічних механізмів.

🎯 Мета роботи
ознайомитися з принципами цифрового підпису;
реалізувати залежність підпису від вмісту файлу та ключів;
дослідити механізм перевірки цілісності даних;
показати неможливість підробки підпису без зміни файлу або ключа.

⚙️ Використані алгоритми

SHA-256 — для створення хешу повідомлення;
RSA (2048 біт) — для генерації ключів;
RSA-PSS — схема створення цифрового підпису.

🧩 Функціональні можливості

Програма дозволяє:
Генерувати пару ключів:
private.pem — приватний ключ;
public.pem — публічний ключ.

Підписувати текстове повідомлення:
створюється SHA-256 хеш;
хеш підписується приватним ключем;
підпис зберігається у файлі signature.b64.

Перевіряти цифровий підпис:
обчислюється новий хеш повідомлення;
перевіряється відповідність підпису;
виводиться результат перевірки.

Програма працює в інтерактивному консольному режимі (через меню).

▶️ Як запустити програму
🔹Вимоги:
Python 3.x
Бібліотека cryptography

Встановлення бібліотеки:
🔹 Запуск
pip install cryptography
Запустіть програму:
python digital_signature_demo.py

Дотримуйтесь інструкцій у консолі.

🧠 Логіка роботи програми
Введене повідомлення хешується алгоритмом SHA-256.
Отриманий хеш підписується приватним ключем RSA.
Підпис зберігається у Base64-форматі.

Під час перевірки:
повторно обчислюється хеш;
публічний ключ перевіряє підпис;
у разі зміни повідомлення підпис стає недійсним.

📊 Приклад роботи
Підписати текст:
Hello

Результат:
Підпис створено.

Перевірити підпис із тим самим текстом:
Підпис ДІЙСНИЙ

Змінити текст (наприклад, Hello!) і перевірити знову:
Підпис НЕДІЙСНИЙ

Це демонструє залежність підпису від вмісту повідомлення.

⚠️ Примітка
Даний алгоритм не є повноцінною реалізацією цифрового підпису і не повинен використовуватися у реальних системах безпеки. Програма призначена виключно для демонстрації базових криптографічних принципів у межах навчального курсу.
