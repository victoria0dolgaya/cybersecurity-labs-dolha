Лабораторна робота №6 Демонстрація SQL-ін’єкції та методів захисту. Етичний хакінг власного застосунку

Дисципліна: Безпека програм та даних

📌 Опис 
Даний застосунок демонструє механізм виникнення SQL-ін’єкції та способи її усунення.
Програма реалізує систему пошуку студентів із використанням локальної бази даних SQLite та графічного інтерфейсу на Tkinter.

У застосунку реалізовано дві версії пошуку:
🔴 Уразлива версія (через конкатенацію SQL-запиту)
🟢 Захищена версія (через параметризований запит)

🎯 Мета роботи 
Ознайомитися з механізмом SQL-ін’єкції, продемонструвати її небезпеку та реалізувати безпечний підхід до роботи з базами даних шляхом використання параметризованих запитів.

⚙️ Функціональні можливості 
Програма дозволяє:
створити тестову базу даних студентів;
виконувати пошук за ім’ям;
продемонструвати успішну SQL-ін’єкцію;
порівняти поведінку уразливої та захищеної версій;
візуально відобразити результати в графічному інтерфейсі.

▶️ Як запустити програму 
Збережіть код у файл, наприклад:
sql_injection_lab6.py

Запустіть програму:
python sql_injection_lab6.py

Після запуску відкриється графічне вікно застосунку.
Демонстрація SQL-ін’єкції

У полі пошуку введіть:
' OR '1'='1
🔴 Уразливий пошук:

Повертає всі записи з бази (витік даних).

🟢 Захищений пошук:
Атака не спрацьовує, оскільки використовується параметризований запит.


⚠️ Примітка Програма має навчальний характер і демонструє базові принципи захисту даних. У реальних системах рекомендується використовувати випадкову сіль та додаткові механізми управління ключами.
